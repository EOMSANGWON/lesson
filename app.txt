**git 가이드북**

[1] git 초기 세팅
    1. git 설치 (윈도우와 맥 버전이 따로 있음)
    2. git을 실행할 폴더 생성, 폴더 우클릭 후 터미널, 
        git config --global user.email "um4own@gmail.com"
        git config --global user.name "엄상원"
    후 빠져나옴

[2] git 실행
    1. 에디터실행, 작업폴더 위치에서 터미널 실행, 
        git init
    을 실행하면 git이 감시를 시작한다. (+repository를 생성)
    git init : 현재 폴더를 git이 관리하는 repository로 초기화한다. .git 이라는 숨김폴더가 생성되고 여기에 버전정보, 커밋 기록등이 저장된다.
    2. 작업 후 저장하기 위해서는 
        git add 파일명
        git commit -m '메모'
    를 터미널에 순서대로 실행시킨다. 기록 완료
    이후 코드를 수정하면 다시 add와 commit을 연달아 실행시킨다.
    git add 파일명 : 선택한 파일의 변경내용을 스냅샷에 담는 행위, staging한다라고도 함
    git add a.txt b.txt : 두 파일을 동시에 스테이징이 가능하다.
    git add . : 모든 파일을 스테이징한다.
    git commit -m '메모': 최종저장하는 행위 (작업단위, 추가나 수정 했을 때 한 번씩 해주는 게 좋음)
    git status : 현재 스테이징 상황 볼수 있음 
    git log --all --oneline : commit 내용을 보여준다

[3] branch  
    1. 복사본을 만듦 
        git branch 새브렌치명
    2. 타 브렌치로 이동
        git switch 브렌치명 **기존의 브랜치는 'main'
    3. 브렌치 로그를 시각적으로 확인가능
        git log --oneline --all --graph

[4] merge 
    1. branch를 합치는 과정
        git merge 브랜치명 : 현재 브랜치와 다른 브랜치를 합친다. 각 브렌치들이 commit이 완성된 상태에서 실행해야함
        main branch와 second branch가 각기 다른 파일들을 수정하고 merge 를 실행하면 잘 이뤄지나...?
